3SCOPE
Scope & Entry Points

Entry Points

Tab "Practice" riêng trên bottom navigation — độc lập hoàn toàn với tab Tiến trình/Streak (C1). Không đọc/ghi streak state.
  • Lưu từ: tap 1 từ trong chat/lesson → bottom sheet
  • Lưu câu: nút 🔖 trên message bubble
  • AI correction: tự xuất hiện inline (không có entry thủ công)

Quan hệ giữa 3 tính năng

Độc lập về data, nhưng TN2 phụ thuộc điều kiện: chỉ có data sau khi hoàn thành ≥1 bài học. TN1 và TN3 có data ngay từ session đầu. Không có thứ tự bắt buộc khi xem.

5FLOW
Flow / Behavior

Flow A — Lưu TỪ chủ động

  1. Tap 1 từ trong message → bottom sheet trượt lên (≤300ms): từ + IPA + nghĩa VI + ví dụ + 🔊 + 🔖 (outline).
  2. Tap 🔖 → optimistic: 🔖 fill ngay + toast "Đã lưu vào Từ đã lưu" (2s), KHÔNG đợi BE.
  3. Gửi POST /me/saved-items {type:"word", content, translation, source_lesson_id, source_session_id}.
  4. BE dedup theo (user_id, type, content) không phân biệt hoa thường → tồn tại: 200 + id cũ; mới: 201.
  5. BE gọi AI enrich async: bổ sung ipa, syllables, example_en, example_vi. Mobile không đợi.
  6. Bỏ lưu: tap lại 🔖 → optimistic về outline → DELETE /me/saved-items/{id}.

Flow B — Lưu CÂU chủ động

Như Flow A nhưng: type:"sentence", không bottom sheet (lưu trực tiếp từ 🔖 trên message), không AI enrich. Dedup theo (user_id, type, content).

Flow C — AI detect lỗi → auto save tự động

  1. User gửi câu (voice/text) → AI đánh giá (rule R4).
  2. Có lỗi → AI trả chat_response + correction inline (< 2s). User thấy ngay.
  3. Async: AI layer gọi POST /me/grammar-errors {...}.
  4. BE INSERT grammar_errors + UPDATE counter: total_errors += 1, errors_by_date[today] += 1, last_error_at = now().
  5. KHÔNG dedup — cùng 1 lỗi N lần = N records.
  6. POST fail → retry tối đa 3 lần. Hết retry → lỗi vẫn hiện chat nhưng KHÔNG vào TN3 (chấp nhận).

Flow D — Curriculum mark "đã học" tự động

  1. Trigger: POST /lesson/complete (C3).
  2. BE query curriculum_items WHERE lesson_id = {completed}.
  3. Mỗi item → tạo user_curriculum_progress {user_id, curriculum_item_id, learned_at: now()}.
  4. Idempotent: đã tồn tại (user_id, curriculum_item_id) → skip.

CMS gắn tag per lesson trước. BE chỉ đọc và đánh dấu.

Flow E — Home card counter

  • TN1: count(type=word) + count(type=sentence)
  • TN2: progress % (R2)
  • TN3: GET /me/error-summary → total_errors

Flow F — Cascade delete

Khi DELETE /me chạy (flow do Login-Auth spec sở hữu — C4), xóa vĩnh viễn 4 bảng: saved_items, user_curriculum_progress, grammar_errors, error_counter. Chỉ đăng ký vào cascade — KHÔNG định nghĩa lại Stripe/Apple/payment.

6LOGIC
Business Rules & Logic

Các rule có thể derive trực tiếp thành dev task và test case.

R1 — Dedup saved_items

Khóa dedup = (user_id, type, LOWER(TRIM(content))). "Hello" và "hello " → cùng 1 item. Không tạo bản sao; trả id đã có.

R2 — Công thức progress % của TN2

Per lesson: progress% = round(số learned / tổng khái niệm CMS × 100). Flow D mark tất cả item khi complete → hoàn thành bài = 100%. Lesson chưa xong không xuất hiện (Q4).

R3 — Pagination

Cursor-based, limit=20/trang. TN1 group theo created_at. TN3 group theo detected_at, mới nhất trên đầu.

R4 — AI Grammar Detection

Scope: chỉ phát hiện lỗi ngữ pháp thực sự (câu SAI). KHÔNG gợi ý cách diễn đạt tự nhiên hơn. Câu đúng ngữ pháp dù không tự nhiên → PASS.

Input / Output Contract

// Input
{ "user_id","session_id","lesson_id?","user_message",
  "conversation_context":[3-5], "user_level", "source":"voice"|"text" }

// Output — có lỗi
{
  "has_error": true,
  "correction": {
    "wrong_sentence": "Let mess it up",
    "correct_sentence": "Let's mess it up.",
    "explanation_vi": "Ở đây cần dùng \"let's\" (viết tắt của \"let us\")...",
    "rule_tag": "let_vs_lets",
    "error_words": ["Let"], "corrected_words": ["Let's"]
  },
  "chat_response": "I get it, but...", "save_to_be": true
}
// Output — không lỗi → { "has_error": false, "correction": null, "save_to_be": false }

Quy tắc ĐÁNH LỖI vs PASS

Hành độngTình huốngVí dụ
ĐÁNH LỖISai thì/cấu trúc/thiếu-thừa từ"I goed" → "I went"
ĐÁNH LỖISai từ trong context"call other car" → "call another car"
ĐÁNH LỖIThiếu article/preposition rõ"I go school" → "I go to school"
PASSĐúng ngữ pháp, informal/ngắn gọn"That good"
PASSSlang phổ biến"Gonna","Wanna","Gotta"
PASSCâu chưa hoàn chỉnh / đang gõ dở"I think..."
PASSTin nhắn quá ngắn (1-2 từ)"Yes", "OK"
PASSCâu hỏi/giao tiếp, không phải bài tập"What do you mean?"

Mức độ sửa theo level

A1/A2 → chỉ sửa lỗi cơ bản (thì, cấu trúc, article). B1+ → sửa chặt hơn (word choice, preposition, collocation). Không overwhelm beginner. Câu nhiều lỗi (3+) → chỉ sửa 1-2 lỗi nghiêm trọng nhất.

Style giải thích VI

100% tiếng Việt (từ EN giữ trong ngoặc kép) · 1-2 câu (tối đa 3) · tone thân thiện như bạn bè · pattern [Ở đây cần...] + [vì...] + [ví dụ đúng]. rule_tag: AI tự sinh snake_case, không phải hard enum, BE lưu nguyên string.

Latency

Detection + correction < 2s (trong chat response) · POST cho BE async, retry 3 lần · IPA enrich < 10s background.

7INV
Critical Invariants
  • Dedup TN1: (user_id, type, LOWER(TRIM(content))) không bao giờ trùng.
  • TN3 KHÔNG dedup: counter = đúng số lần mắc thực tế.
  • Flow D idempotent: (user_id, curriculum_item_id) duy nhất.
  • AI→BE async KHÔNG chặn chat: correction hiện trước; fail → retry ≥3.
  • total_errors chỉ tăng, không reset.
  • Cascade delete 4 bảng cùng DELETE /me.
  • Detail sheet KHÔNG có mic. AI KHÔNG gợi ý cách diễn đạt.
  • Practice Hub KHÔNG đọc/ghi streak state (tab độc lập).
